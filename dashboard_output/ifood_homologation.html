6. Reviews - Ciclo de Homologação Req Reviews: 4 testes ▼
Checklist iFood Reviews v2.0: listagem com paginação e filtro de data, detalhes completos, resposta com validação de status/tamanho, resumo com score. Documentação obrigatória: Política de Avaliações do iFood (link deve estar visível na UI final).
6.1 Listar Avaliações — GET /reviews
Validações: Status 200 · campos obrigatórios (id, status, replies[], version, visibility) · paginação (page, size, total, pageCount) · pageSize > 50 retorna 400 · reviews:[] quando vazio.
6.2 Obter Detalhes — GET /reviews/{reviewId}
Validações: Status 200 · todos os campos v2 presentes · replies[].from ∈ {MERCHANT, CUSTOMER} · ID inexistente retorna 404.
6.3 Responder — POST /reviews/{reviewId}/answers
Só aceita reviews com status: NOT_REPLIED. Texto entre 10 e 300 caracteres.
Validações: Status 201 com resposta válida · 400 para texto fora do intervalo · 409/422 se review já foi respondida (PUBLISHED).
6.4 Obter Resumo — GET /summary
Validações: totalReviewsCount ≥ 0 · validReviewsCount ≤ totalReviewsCount · score ∈ [1.0, 5.0] · 404 "Summary not found" é estado válido quando a loja ainda não tem reviews agregados.
